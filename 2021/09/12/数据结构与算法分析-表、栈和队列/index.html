    1.3.4 rule of zero / rule of five

    • rule of zero （通常在类中有指针时不满足，则需要遵循rule of five）
      如果类中没有自己定义除了默认构造函数之外的特殊成员函数，那么就不应该定义任何一个拷贝/移动构造函数、拷贝/移动operator=和析构函数。由编译器自动合成上述函数。
    • rule of five

    拷贝/移动构造函数、拷贝/移动operator=和析构函数这五个函数中显示定义了一个，则其他的也应该显示定义。

    1.3.5 C++11对拷贝构造函数和拷贝operator=的简化和安全处理

    • 对拷贝构造函数，可以使用构造函数转发，将初始化基础成员变量和分配内存的工作委托给构造函数 
      Vector(const Vector& rhs) : Vector{rhs.theSize} {
 for(int i=0; i<theSize; i++) {
     objects[i] = rhs.objects[i];
 }
}
    • 对拷贝operator=，可以使用copy-and-swap技术
      ```cpp
      friend void swap(Vector& a, Vector& b) {
      using std::swap;
      swap(a.theSize, b.theSize);
      swap(a.theCapacity, b.theCapacity);
      swap(a.objects, b.objects);
      }

    Vector& operator=(Vector rhs) {
    swap(*this, rhs);
    return *this;
    }

    ### 1.3.6 下标运算符[]的两个版本  
成员函数的**定常性**时签名的一部分，因此可以构建**访问版本**的`[]`和**修改版本**的`[]`  
```cpp
// 修改版本，返回引用
Object& operator[] (int index) {
    return objects[index];
}  
// 访问版本，返回const引用
const Object& operator[] (int index) const {
    return objects[index];
}

    当对const的Vector使用下标运算符时，会调用访问版本，返回常引用（不会被修改）。
